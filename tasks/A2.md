You are assisting with project management analysis. Follow the output format exactly.

## Task

A software team is launching a customer portal. Durations are in working days. A task can only start once all of its predecessors are finished.

| Task | Name | Duration (days) | Depends on |
|---|---|---|---|
| T1 | Requirements workshop | 3 | none |
| T2 | API design | 4 | T1 |
| T3 | UI design | 5 | T1 |
| T4 | Backend build | 8 | T2 |
| T5 | Frontend build | 6 | T3 |
| T6 | Integration | 3 | T4, T5 |
| T7 | QA testing | 5 | T6 |
| T8 | Documentation | 2 | T6 |
| T9 | Deployment | 1 | T7, T8 |

Compute the critical path, the total project duration, and the total slack (float) of every task in days.

## Output format

Respond with a single JSON object and nothing else. No prose before or after, no code fences.

{
  "critical_path": ["T1", "..."],
  "project_duration_days": 0,
  "slack_days": {"T1": 0, "T2": 0, "T3": 0, "T4": 0, "T5": 0, "T6": 0, "T7": 0, "T8": 0, "T9": 0}
}

The critical path must list task IDs in order from first to last.
