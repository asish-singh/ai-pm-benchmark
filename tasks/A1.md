You are assisting with project management analysis. Follow the output format exactly.

## Task

An office renovation project has the following tasks. Durations are in working days. A task can only start once all of its predecessors are finished.

| Task | Name | Duration (days) | Depends on |
|---|---|---|---|
| T1 | Demolition | 5 | none |
| T2 | Electrical rough in | 4 | T1 |
| T3 | Plumbing rough in | 6 | T1 |
| T4 | Drywall | 5 | T2, T3 |
| T5 | Painting | 3 | T4 |
| T6 | Flooring | 4 | T4 |
| T7 | Fixtures install | 2 | T5, T6 |
| T8 | Final inspection | 1 | T7 |

Compute the critical path, the total project duration, and the total slack (float) of every task in days.

## Output format

Respond with a single JSON object and nothing else. No prose before or after, no code fences.

{
  "critical_path": ["T1", "..."],
  "project_duration_days": 0,
  "slack_days": {"T1": 0, "T2": 0, "T3": 0, "T4": 0, "T5": 0, "T6": 0, "T7": 0, "T8": 0}
}

The critical path must list task IDs in order from first to last.
